# Comet Browser - Advanced Production Guide v0.5.2

## ï¿½ Full Setup & Deployment Process

### 1. Repository Preparation
- Clone the repository to your local machine or server.
- Install dependencies: `npm install`
- Copy `env.example` to `.env`: `cp env.example .env`

### 2. Environment Configuration (.env)
Open `.env` and configure the following critical sections:
- **Identification**: Set `NEXT_PUBLIC_APP_NAME` to your preferred browser name.
- **Intelligence**: Add your OpenAI, Gemini, or Claude API keys for cloud features. Local processing (TF.js) works out of the box.
- **Security (Firebase)**: Setup a Firebase project at [console.firebase.google.com](https://console.firebase.google.com). Enable "Google" as an Auth provider. Copy the Web App config into the `NEXT_PUBLIC_FIREBASE_*` variables.
- **Sync Core (MySQL)**: Set up a MySQL database for device-to-device synchronization. The `MYSQL_URL` is used for global state distribution.
- **Distribution (Downloads)**: Once you build your Electron installers, host them and put the direct links in `NEXT_PUBLIC_WINDOWS_DOWNLOAD_URL`, etc.

### 3. Development Mode
Run the Next.js dev server:
`npm run dev`
Run Electron in a separate terminal:
`npm start`

### 4. Admin Portal Access
- Navigate to `/admin` in your browser.
- Log in with Google.
- **Note**: The default Master Admin is `preetjgfilj2@gmail.com`. You can change this in `src/app/admin/page.tsx` for your own deployment.

### 5. Deployment (Vercel)
- Push your code to GitHub.
- Connect the repo to Vercel.
- **IMPORTANT**: In Vercel Project Settings, add ALL variables from your `.env` to the "Environment Variables" section.
- Deploy. Your landing page will be live at your Vercel URL.

### 6. Building Native Apps (Electron)
- For Windows: `npm run build:win`
- For Mac: `npm run build:mac`
- For Linux: `npm run build:linux`
- Upload the resulting files in the `dist` folder to your file hosting service and update your `.env` download links.
